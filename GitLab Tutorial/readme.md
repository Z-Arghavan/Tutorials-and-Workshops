# A Quick Guide to GitLab at TU/e

This guide is for the ISBE students to create an account, configure a repository, upload their project to GitLab, and overcome any issus or challenges.

---

## 1. Login

- Go to GitLab: https://gitlab.tue.nl/users/sign_in  
- Click **“Login with TU/e”** 

If it does not work:
- Try another browser or check the cache and cookies
- If still not working, send email to gitlab@tue.atlassian.net  

---

## 2. Create or Access a Repository

- If you don’t have one → click **“New Project”**  
- If you already have one → ask your supervisor for access  

---

## 3. Upload Your Work

- Upload your files
- Add a short description (README)
- README uses markdown language. Look at markdown cheatsheet: https://www.markdownguide.org/cheat-sheet/

At minimum include:
- Project name  
- What it does  
- Your name  

Click **Commit** to save

---

## 4. Make It Public (if needed)

- Go to **Settings → Visibility**  
- Choose:
  - Internal (safe default)  
  - Public (only if allowed)

If you cannot change it, send email to gitlab@tue.atlassian.net  

---

## 5. What You Can Share

You can upload:
- Code  
- Demo / prototype  
- Short description
-  Reports

Do NOT upload:
- Full paper before publication  

---

## 6. Need Help?

- Login problem → gitlab@tue.atlassian.net  
- Cannot access repo → gitlab@tue.atlassian.net  
- Anything else → ask your supervisor or gitlab@tue.atlassian.net
